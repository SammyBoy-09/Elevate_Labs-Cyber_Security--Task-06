Task - 06

Password Strength Evaluation Report

Objective: Evaluate varying password complexities and establish best practices for secure authentication.

Password Analysis

The following passwords were evaluated based on length, complexity (character sets used), and predictability (resistance to dictionary attacks).

Sample 1: Short & Predictable Very Weak

summer2024

Analysis: This password is highly vulnerable. It uses a common dictionary word ("summer") followed by a predictable year. It lacks uppercase letters and special symbols.

Attack Vector: Easily defeated by a basic dictionary attack or a hybrid attack within seconds.

Figure 1: Password checker indicating "summer2024" can be cracked in roughly 25 seconds.

Sample 2: Complex but Predictable Medium

T3chn0l0gy#89

Analysis: This password is 13 characters long and includes all four character sets. However, it relies heavily on standard "leet speak" substitutions for a single dictionary word, which modern cracking tools automatically check.

Attack Vector: Will defeat basic online brute-forcing but remains vulnerable to advanced offline dictionary attacks using rulesets.

Figure 2: Example of how common substitutions (like P@ssw0rd1) are instantly recognized by checkers and cracking tools, resulting in a crack time of 0 seconds.

Sample 3: Length & Complexity (Passphrase) Very Strong

Blue^Guitar#Running$Quickly99!

Analysis: This is a "passphrase" containing 30 characters, utilizing all four character sets randomly distributed. Length is the most critical factor in password entropy.

Attack Vector: Highly resistant to both dictionary and brute-force attacks. The sheer length makes the computational time required to guess it astronomical.

Figure 3: Password checker indicating the long passphrase would take an estimated 8 billion years to crack.

Best Practices & Takeaways

  • Length is Paramount: A 16-character password with just lowercase letters is often stronger than an 8-character password with high complexity. Aim for 14+ characters or use passphrases.
  • Avoid Predictability: Never use personal information (names, birthdates), sequential characters ("123456", "qwerty"), or common words. Avoid simple "leet speak" substitutions.
  • Use Password Managers: Utilize tools like Bitwarden or 1Password to generate, store, and auto-fill complex, unique passwords for every account.
  • Enable MFA: A strong password is only one layer of defense. Multi-Factor Authentication (MFA) is essential to protect accounts even if the password is compromised.
